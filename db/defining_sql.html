Commonly used data types

CHAR(n)

Character string of fixed length n.

VARCHAR(n)

Character string of variable length (with maximum length n).

BOOLEAN

TRUE or FALSE

INT

Integer

FLOAT

Number with floating decimal point.

DATE

#DD/MM/YYYY#

TIME

HH:MM:SS

CURRENCY


Defining a table

Using CREATE TABLE

Example:

CREATE TABLE Employee
(
EmpID       INTEGER NOT NULL, PRIMARY KEY,
EmpName     VARCHAR(20) NOT NULL,
HireDate    DATE,
Salary      CURRENCY
)

NOT NULL

Means that the field defined must be given a value.

PRIMARY KEY

Sets the defined field to the primary key.

Altering table structure

Adding a column

ALTER TABLE Employee
ADD Department VARCHAR(10)

Deleting a column

ALTER TABLE Employee
DROP COLUMN HireDate

Changing the data type of a column

ALTER TABLE Employee
MODIFY COLUMN EmpName VARCHAR(30) NOT NULL

Linking tables with foreign keys

Employee table (link 1)

EmpID Integer, primary key
Name
HireDate
Salary
Department

Course table (link 2)

CourseID 6 characters, fixed length, primary key
CourseTitle
OnSite

CourseAttendance table (links the above tables)

CREATE TABLE CourseAttendance
(
CourseID        CHARACTER(6) NOT NULL,
EmpID           INTEGER NOT NULL,
CourseDate      DATE,
FOREIGN KEY     CourseID REFERENCES Course(CourseID),
FOREIGN KEY     EmpID REFERENCES Employee(EmpID),
PRIMARY KEY     (CourseID, EmpID)
)

Inserting a new record

Syntax

INSERT INTO table(col1, col2, ...)
VALUES (val1, val2, ...)

If all fields are being added to, and in the correct order, only INSERT INTO is needed.

Updating a record

Syntax

UPDATE table
SET col1 = val1, col2 = val2, ...
WHERE colx = val

Deleting a record

Syntax

DELETE FROM table
WHERE colx = val
