Frequently Asked Questions

How do I ssh into the CS server?

Type ssh tinky-winky.cs.bham.ac.uk and input your username and password.

Why can't I use gedit over ssh?

You can (put -X after ssh in the command above), but you shouldn't (see below). The reason you can't without -X is that gedit is a graphical application, and the only thing being directed through ssh is the terminal screen you see.

Why shouldn't I direct the display server through ssh?

Because it's sending all of that window information over the internet, and that's gonna be laggy and unusable.

What can I use for editing my files on the CS server then?

  1. scp (copy the files over via ssh) (See this page for more info)
  2. sshfs (setup a virtual ssh filesystem so you can access your stuff with your native applications) (See this page for more info)
  3. Use git, it's really good at doing this stuff and you'll need to learn it at some point. Create an account at gitlab.com or github.com and set up a new repository. (See this page for git commands and some brilliant explanations)

This right here is crazy talk. Why can't I just use a USB like everyone else?

  1. You'll have to remember to transfer everything over every time you work.
  2. Everyone loses their USBs by leaving them plugged in.
  3. You're a CS student and deserve more efficient ways to improve your workflow.
  4. Bragging rights.
  5. You'll probably have to use one of the above at some point and it will be under practical circumstances, so do it now.
